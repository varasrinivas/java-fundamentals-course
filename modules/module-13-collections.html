Lists, Sets and Maps, and when to pick which

Java ships a small, well-worn set of data structures behind three interfaces: List, Set, and Map. Knowing how HashMap finds a key in O(1) — and when it degrades — is core Java literacy.

Concept

Three interfaces, a handful of workhorse classes

Program to the interface, choose the implementation for its performance: List (ordered, indexed — usually ArrayList), Set (no duplicates — HashSet), Map (key→value — HashMap).

A HashMap turns a key into an array index: take the key's hashCode(), spread its bits, and reduce modulo the bucket count. Two keys landing in one bucket is a collision, handled by a short chain (which becomes a tree if it grows). When the table gets too full it resizes and rehashes.

Coming from another language

    
What will surprise you

      
Coming from Python
HashMap ≈ dict, ArrayList ≈ list, HashSet ≈ set. Big difference: Java's HashMap does not preserve insertion order — use LinkedHashMap for that (Python dicts are ordered by default).

      
Coming from JavaScript
Java's Map is a real interface with many implementations; JS's Map is one built-in. And Java keys use equals()/hashCode(), so two distinct objects that are equal collide as the same key — unlike JS's identity-based Map keys.

      
Coming from C#
Dictionary<K,V> ≈ HashMap, List<T> ≈ ArrayList. Java separates the interface (Map) from the class (HashMap) more strictly, so you'll often declare the interface and instantiate the class.

  






  

    
Exercises

    
Make it stick

      
1 · Pick the structure
You need to remember which user IDs you've already emailed, and ask “did I email 4187?” millions of times. Which collection?
💡 Show hint
A HashSet<Integer> (or a Set). Membership tests are O(1) average, and you don't care about order or duplicates — exactly a Set's job.

      
2 · The broken key
You use a mutable object as a HashMap key, then change a field used in its hashCode(). Why can you no longer find it?
💡 Show hint
Its hash now points to a different bucket than where it was stored, so get looks in the wrong place. Map keys must be effectively immutable in their hashed fields.
