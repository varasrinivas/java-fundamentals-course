A factory for matching families

Real-world picture: a furniture showroom. Order from the “Victorian” catalogue and the sofa, chair, and table all match. Order “Modern” and you get a different matching set. Abstract Factory creates whole families of related objects that are guaranteed to go together — without naming their concrete classes.

Concept

One factory, a coordinated set of products

Factory Method (module 35) makes one product. Abstract Factory makes a family. You define a factory interface with several create-methods, and each concrete factory produces a matching set. Classic use: UI toolkits where every widget must match the OS look.

Pick the factory once and the whole app produces a consistent family — no risk of a Mac button next to a Windows checkbox. The trade-off: adding a new product means changing every factory. Reach for it when products come in interchangeable, themed sets.

Coming from another language

    
What will surprise you

      
Coming from Python
Often a factory class (or even a module) whose functions return a matching set — duck typing removes the interface ceremony. The “pick one provider, get a consistent family” idea is identical.

      
Coming from JavaScript
A factory object whose methods return matching parts (e.g. a theme object that builds matching components). No interface keyword, same coordination guarantee.

      
Coming from C#
Nearly identical; DI containers frequently act as abstract factories, resolving a coordinated set of services for one configuration.

  






  

    
Exercises

    
Make it stick

      
1 · Factory Method vs Abstract Factory
What’s the core difference between Factory Method (35) and Abstract Factory?
💡 Show hint
Factory Method creates one product via a single overridable method. Abstract Factory creates a family of related products via an object with several create-methods, guaranteeing the products match each other.

      
2 · The trade-off
Adding a new product type (say a Slider) to an Abstract Factory is painful. Why?
💡 Show hint
Every concrete factory must implement the new slider() method, so you touch them all. Abstract Factory makes adding a new family easy but adding a new product hard — the opposite of what you might expect.
