A class is a blueprint; new stamps out objects

In Java, almost everything you build is a class with fields, a constructor, and methods. There are no free functions and no top-level code — understanding the new→allocate→construct sequence is understanding how Java programs come alive.

Concept

From class definition to living object

A class declares state (fields) and behavior (methods). A constructor runs once, right after allocation, to put a new object into a valid initial state. The new operator does three things in order: allocate heap memory, run the constructor, hand back a reference.

Fields default to zero / false / null before the constructor runs. The constructor's job is to replace those defaults with meaningful values — ideally leaving no half-built object observable.

Coming from another language

    
What will surprise you

      
Coming from Python
No self as an explicit first parameter — Java uses an implicit this. And Java's constructor is named after the class, not __init__. Fields are usually declared up front, not created ad-hoc in the constructor.

      
Coming from JavaScript
Java classes aren't sugar over prototypes — they're the real, only model. There's no object literal {}; you must define a class (or a record) first. Visibility (private) is enforced by the compiler, not a # convention.

      
Coming from C#
Very close: fields, constructors, this, and new all match. Java lacks C# properties (get; set;), so you'll write explicit getter/setter methods or use records.

  






  

    
Exercises

    
Make it stick

      
1 · Default before construct
A class has an int count; field and a constructor that doesn't touch it. What is count right after new?
💡 Show hint
0. Numeric fields default to 0, booleans to false, and object references to null — before and unless the constructor overrides them.

      
2 · Why private?
Why make balance private and expose deposit() instead of letting callers set it directly?
💡 Show hint
Encapsulation: the class controls its invariants. deposit() can validate (reject negatives), log, or notify. A public field gives callers raw access with no guardrails.
